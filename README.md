# zozyaz

Проверка текста email на соответствие 168-ФЗ через ГосСловарь API.

## Как работает проверка

1. Если во входе HTML, приложение извлекает видимый текст.
2. Текст режется на чанки по `650` символов (без overlap).
3. Каждый чанк отправляется в `POST https://gosslovar.ru/api/check/text`.
4. Результат чанка опрашивается отдельными запросами `GET /api/results/text?id=...`.
5. Между стартами чанков добавляется задержка `100-500ms`.
6. Все `missing_words` склеиваются обратно в глобальные позиции и показываются в UI.

## API ключ

- Ключ вводится в поле `API ключ ГосСловарь` на вкладке проверки.
- Ключ сохраняется локально в `localStorage` (`gosslovar_api_key`).
- Ключ не хранится в репозитории.

## Ограничения

- Ручной `imageText` отключен в API-режиме.
- OCR в интерфейсе скрыт и отложен на следующий этап.
- При ошибке API показывается статус ошибки (fallback на локальную словарную проверку отключен).

## Важный нюанс CORS

Проверка выполняется из браузера напрямую в `https://gosslovar.ru/api/...`.  
Для production это требует корректных CORS-заголовков со стороны API для вашего origin.

Если браузер в DevTools показывает блокировку preflight/response по CORS:

- обратитесь к поставщику API для разрешения origin;
- либо используйте прокси/другой способ доставки запросов.
